Case 1 — 자사몰

case 1-1-I  ·  자사몰 신규 회원가입

처음 가입하는 사용자가 이메일/비밀번호로 회원가입 + 프로필 작성

1사용자가 회원가입 페이지 진입
이동: /signup
step 1 폼 표시 (이메일 + 비밀번호 + 확인)
src/app/signup/_components/SignupClient.tsx
↓
2이메일 / 비밀번호 입력 후 "다음" 클릭
checkEmailExists(email) 호출 → GET /api/v1/veggieverse/users/email-check
자사몰 DB에 이메일 존재 여부 사전 확인
handleStep1Submit · src/lib/api/user.ts
↓
?email-check 결과 분기
exists = true
AlreadyRegisteredModal 표시 → case 2-1-II로 진입
exists = false
다음 단계로 진행 (supabase 가입)
↓
3supabase 계정 생성 (server action)
signUpAction({ email, password }) → supabase.auth.signUp(...)
성공 시 backend POST /auth/token 으로 JWT 발급
src/app/auth/actions.ts · handleStep1Submit
↓
4step 2 폼 표시 (프로필 정보)
이름 · 휴대전화 · 생년월일 · 주소 · 프로필 이미지 · 약관 동의 입력
이름 필드: 한글/영문 + IME 조합용 자모(ㄱ-ㅎ ㅏ-ㅣ) 허용
↓
5"회원가입" 클릭
① supabase.auth.updateUser({ data: { full_name } }) — user_metadata 갱신
② POST /auth/token 재발급 (name claim 포함된 JWT 확보)
③ POST /api/v1/veggieverse/users/profile (multipart) — 백엔드 ConsumerUser INSERT + 이미지 업로드
handleStep2Submit
↓
✓회원가입 완료
refetchProfile() 호출 → 헤더가 새 프로필 즉시 fetch
"회원가입이 완료되었어요" 페이지 표시

1"카카오로 계속하기" 클릭

카카오 → /auth/callback?code=...
↓

↓

+ email-check OK
→ /signup?prompt=existing-email
↓

isExistingEmailPrompt useEffect 발동 → 이메일 자동 채움 + 모달 open
SignupClient · AlreadyRegisteredModal (mode=from-kakao-flow)

이 이메일은 자사몰에 이미 가입되어 있습니다.
자사몰 계정으로 '이메일로 로그인'하거나, 자사몰 계정에 카카오를 추가하려면 '카카오 연동하기'를 선택해 주세요.
K 카카오 연동하기 이메일로 로그인 취소
↓

case 1-2  ·  자사몰 로그인

기존 이메일/비밀번호로 로그인

1로그인 모달 또는 /login 에서 email/pw 입력
LoginModal · LoginClient
↓
2signInAction 호출 (server action)
supabase.auth.signInWithPassword({ email, password })
성공 시 결과 + session tokens(access_token, refresh_token) 반환
⚠️ Next.js 16 + Turbopack dev에서 server action redirect의 client navigation hang을 피하기 위해 redirect 대신 결과 반환
src/app/auth/actions.ts
↓
3client에서 setSession 명시 호출
supabase.auth.setSession({ access_token, refresh_token }) → onAuthStateChange("SIGNED_IN") 발화 → UserContext 즉시 갱신
LoginModal · LoginClient handleSubmit
↓
✓로그인 완료
LoginModal: onClose() + router.refresh() (현재 페이지 유지)
LoginClient: router.push("/") (홈으로)

Case 2 — 카카오

case 2-1-I  ·  카카오 신규 회원가입

자사몰에 가입된 적 없는 사용자가 카카오로 처음 가입

1"카카오로 계속하기" 클릭
signInWithKakaoAction → 카카오 OAuth URL → window.location.href
↓
2카카오 OAuth → 동의
카카오 → /auth/callback?code=...
↓
3callback — supabase 세션 교환 + 분기 판단
exchangeCodeForSession(code)
user.identities = [kakao only] → hasEmailIdentity = false
email-check 호출 → fail (자사몰 DB에 이메일 없음)
↓
4/signup 으로 redirect (link 파라미터 없음)
이동: /signup
↓
5SignupClient 자동 setStep(2)
isLoggedIn=true && !isLinkMode && !isExistingEmailPrompt → 1단계 폼 패스, step 2 직행
카카오 user_metadata.name 으로 이름 필드 자동 채움
↓
6프로필 입력 + "회원가입" 클릭
① supabase.auth.updateUser({ data: { full_name } })
② supabase.auth.refreshSession() — 갱신된 metadata가 새 access_token에 반영
③ POST /users/profile (auth: required, proxy가 갱신된 supabase 토큰 forward)
↓
✓회원가입 완료
refetchProfile() + 완료 페이지 표시

case 2-1-II  ·  카카오 가입자가 자사몰 회원가입 시도

카카오로 가입된 사용자가 같은 이메일로 자사몰 회원가입 페이지 진입

1/signup 에서 email/pw 입력 후 "다음"
checkEmailExists(email) → 자사몰 DB에 이메일 OK
↓
2AlreadyRegisteredModal 표시 (mode=from-email-flow)
📩 이미 가입된 이메일입니다
카카오로 가입하셨다면 '카카오로 로그인'을, 자사몰 계정으로 연동하려면 '이메일 연동하기'를 선택해 주세요.
K 카카오로 로그인 이메일 연동하기 취소
↓
?사용자 선택
카카오로 로그인
sessionStorage[PENDING_KAKAO_LOGIN_KEY] = timestamp
→ 카카오 OAuth → callback → mount 시 flag 발견 → router.replace("/")
이메일 연동하기
sessionStorage[PENDING_LINK_PASSWORD_KEY] = {password, ts}
→ 카카오 OAuth → callback → /signup?prompt=existing-email → flag 보고 /signup?link=1 로 redirect
취소
모달만 닫음 (사용자가 step 1 폼에 머무름)
↓
4[이메일 연동] /signup?link=1 진입
sessionStorage의 비밀번호를 자동으로 폼에 채움 (TTL 2분 검증)
사용자는 "연동하기" 버튼만 클릭하면 됨
↓
5[이메일 연동] "연동하기" 클릭
linkPasswordAction(linkPassword) → supabase.auth.updateUser({ password })
카카오 supabase 계정에 비밀번호 identity 추가
handleLinkSubmit
↓
✓연동 완료
sessionStorage 비밀번호 정리 → refetchProfile() + router.push("/")
💡 TTL 검증: PENDING_LINK_PASSWORD_KEY · PENDING_KAKAO_LOGIN_KEY 모두 2분 TTL. OAuth 취소 등으로 잔존한 flag는 다음 진입 시 만료 검사로 무시되어 의도 외 동작 방지.
