## A. 인증 — 이메일 인증 + token_hash 전환 · `1af617c`, `dca6a30`

### A-1. 회원가입/비번재설정 (`1af617c`)
- 이메일/비번 회원가입에 Supabase **이메일 인증(Confirm email)** 도입
  - signUp 후 "인증 메일 발송" 안내 → 메일 링크 인증 → step2(프로필)
- **PKCE(`?code=`) → token_hash + verifyOtp(`/auth/confirm`)** 로 전환
  - Server Action 환경의 "code verifier not found" 회피, 쿠키를 redirect 응답에 직접 부착
  - 인증 후 원래 탭이 `/auth/session-check` 폴링으로 세션 감지 → 자동 step2 진입
- 중복 이메일 처리 강화
  - `checkEmailExists`가 providers 반환 → 이메일/카카오 계정 구분 모달 분기
  - enumeration 보호 ON 대응(빈 identities로 기존 계정 판별)
  - 재가입 시 로그인 먼저 시도 → 인증완료·step2 미완료는 메일 재발송 없이 step2 이어감
  - email-check rate-limit 시 하드 블록 대신 진행(graceful degradation)
- 비밀번호 재설정도 token_hash(`/auth/confirm`, `type=recovery`)로 전환

### A-2. 로그인 빌드 수정 (`dca6a30`)
- `LoginClient`의 `?email=` prefill(`useSearchParams`)이 정적 prerender에서 CSR bailout으로 빌드를 깨뜨림 → signup과 동일하게 `Suspense`로 감쌈

## E. 구독 플래너 — 전면 개편 · `2e51406`

### E-2. 가변 플랜 + 캘린더 제거
- 달력(주간 이동·요일·공휴일·날짜) 제거 → **"1일차 ~ N일차"** 구조
- 플랜 길이 **5/7/10/14일**, 하루 **1/2끼** 선택. `totalSlots = 일수 × 끼니`(동적)
- 내부 순차 날짜는 백엔드 payload(`date`)·slotId 호환용으로만 유지(UI 미노출)
- 길이/끼니 변경 시 담긴 메뉴는 일차 기준 보존(가격·개수는 유효 슬롯만 계산)

### E-3. 담는 방식
- 카드 클릭 = 상세 모달, 모달 안 "담기" 버튼으로 담기

- 슬롯 선택 시 메뉴 영역 상단 **안내 배너 + 대상 하이라이트**
- `selectSlot` 랜덤 자동채움 제거 → 순수 선택 토글
