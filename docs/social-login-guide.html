1. 전체 아키텍처 — OAuth 2.0 Code Flow + PKCE

세 공급자(Naver, Google, Kakao)는 모두 OAuth 2.0 Authorization Code Flow를 따릅니다. 브라우저(SPA)에서 직접 토큰을 교환하면 client_secret이 노출되므로, 모든 토큰 교환은 Next.js 서버 라우트(Route Handler)에서 수행해야 합니다. 프론트의 역할은 “로그인 시작 버튼”과 “로그인 완료 후 UI 갱신”뿐입니다.

핵심 원칙

① client_secret은 절대 브라우저 번들에 포함하지 않는다 (NEXT_PUBLIC_ 접두사 금지).
② state(CSRF 방어) + nonce(replay 방어) + PKCE code_verifier는 서버 측 쿠키에 임시 저장.
③ 세션 쿠키는 반드시 HttpOnly · Secure · SameSite=Lax로 발급한다.

공통 플로우

[Browser] /login 페이지에서 "OOO 로그인" 클릭
   │  <a href="/api/auth/{provider}/start">
   ▼
[Next.js] /api/auth/{provider}/start  (Route Handler, GET)
   │  · state, nonce, code_verifier 생성
   │  · 임시 쿠키 3종(HttpOnly, 5분 TTL) 저장
   │  · 공급자 인가 URL로 302 redirect
   ▼
[Provider 로그인 화면] 사용자가 동의
   │
   ▼
[Next.js] /api/auth/{provider}/callback?code=...&state=...
   │  · state 일치 검증 (쿠키 vs 쿼리)
   │  · code + code_verifier → access_token 교환 (서버↔서버)
   │  · access_token → 사용자 정보 조회
   │  · 우리 DB에 upsert → 세션 쿠키 발급
   │  · 임시 쿠키 3종 삭제
   ▼
[Browser] redirect_to(원래 가려던 페이지) 로 redirect

왜 PKCE를 쓰는가?

PKCE(Proof Key for Code Exchange)는 본래 모바일/SPA용이지만, 서버 사이드 OAuth에서도 권장됩니다. 인가 코드가 중간에 탈취돼도 code_verifier를 모르면 토큰 교환이 불가능합니다. Google과 Kakao는 표준 PKCE를 지원하고, Naver는 state만 필수입니다(이 가이드는 모두에 PKCE 적용 가능한 형태로 작성).

2-2. 공통 OAuth 유틸

state·nonce·PKCE 생성을 한 곳에 모아 둡니다. Web Crypto API만 사용하므로 Edge·Node 양쪽에서 동작합니다.

src/lib/oauth.ts
import "server-only";

const enc = new TextEncoder();

function base64url(bytes: ArrayBuffer | Uint8Array): string {
  const b = bytes instanceof Uint8Array ? bytes : new Uint8Array(bytes);
  let s = "";
  for (const x of b) s += String.fromCharCode(x);
  return btoa(s).replace(/\+/g, "-").replace(/\//g, "_").replace(/=+$/, "");
}

export function randomString(byteLen = 32): string {
  const buf = new Uint8Array(byteLen);
  crypto.getRandomValues(buf);
  return base64url(buf);
}

export async function pkceChallenge(verifier: string): Promise<string> {
  const hash = await crypto.subtle.digest("SHA-256", enc.encode(verifier));
  return base64url(hash);
}

export const TEMP_COOKIE_OPTS = {
  httpOnly: true,
  secure: process.env.NODE_ENV === "production",
  sameSite: "lax" as const,
  path: "/",
  maxAge: 60 * 5, // 5분
};

2-3. 세션 쿠키 발급/검증 (Jose 사용)

인증 라이브러리를 도입하지 않고 직접 JWT 세션을 굴리는 가장 가벼운 패턴입니다. Auth.js(NextAuth) v5를 쓰는 선택지도 있지만, Cafe24 SSO 등 커스텀 세션이 필요한 우리 구조에서는 직접 구현이 더 명확합니다.

yarn add jose
src/lib/session.ts
import "server-only";
import { cookies } from "next/headers";
import { SignJWT, jwtVerify } from "jose";

const SESSION_COOKIE = "vv_session";
const key = new TextEncoder().encode(process.env.SESSION_SECRET!);

export type SessionPayload = {
  userId: string;
  provider: "naver" | "google" | "kakao";
  email?: string;
  name?: string;
};

export async function createSession(payload: SessionPayload) {
  const token = await new SignJWT(payload)
    .setProtectedHeader({ alg: "HS256" })
    .setIssuedAt()
    .setExpirationTime("7d")
    .sign(key);

  const store = await cookies(); // Next.js 16: async!
  store.set(SESSION_COOKIE, token, {
    httpOnly: true,
    secure: process.env.NODE_ENV === "production",
    sameSite: "lax",
    path: "/",
    maxAge: 60 * 60 * 24 * 7,
  });
}

export async function readSession(): Promise<SessionPayload | null> {
  const store = await cookies();
  const token = store.get(SESSION_COOKIE)?.value;
  if (!token) return null;
  try {
    const { payload } = await jwtVerify(token, key, { algorithms: ["HS256"] });
    return payload as unknown as SessionPayload;
  } catch {
    return null;
  }
}

export async function destroySession() {
  const store = await cookies();
  store.delete(SESSION_COOKIE);
}

Next.js 16의 변경 포인트

· cookies()가 async입니다. 반드시 await cookies()로 호출하세요.
· middleware.ts는 deprecated되어 proxy.ts로 이름이 바뀌었습니다 (뒤 §5에서 사용).

  5. 서비스 URL: https://veggieverse.com
  6. Callback URL: https://veggieverse.com/api/auth/naver/callback (개발용으로 http://localhost:3000/...도 함께 등록)

로그인 시작 라우트

사용자가 /api/auth/naver/start에 GET하면 state를 발급해 쿠키에 저장하고 네이버 인가 URL로 302 리다이렉트합니다.

src/app/api/auth/naver/start/route.ts
import { NextRequest, NextResponse } from "next/server";
import { cookies } from "next/headers";
import { randomString, TEMP_COOKIE_OPTS } from "@/lib/oauth";

export async function GET(req: NextRequest) {
  const state = randomString();
  const next = req.nextUrl.searchParams.get("next") ?? "/";

  const url = new URL("https://nid.naver.com/oauth2.0/authorize");
  url.searchParams.set("response_type", "code");
  url.searchParams.set("client_id", process.env.NAVER_CLIENT_ID!);
  url.searchParams.set("redirect_uri", process.env.NAVER_REDIRECT_URI!);
  url.searchParams.set("state", state);

  const store = await cookies();
  store.set("oauth_state_naver", state, TEMP_COOKIE_OPTS);
  store.set("oauth_next", next, TEMP_COOKIE_OPTS);

  return NextResponse.redirect(url);
}

콜백 라우트

네이버가 code와 state를 붙여 돌아옵니다. 검증 → 토큰 교환 → 사용자 정보 조회 → 세션 발급 순서.

src/app/api/auth/naver/callback/route.ts
import { NextRequest, NextResponse } from "next/server";
import { cookies } from "next/headers";
import { createSession } from "@/lib/session";

export async function GET(req: NextRequest) {
  const code = req.nextUrl.searchParams.get("code");
  const state = req.nextUrl.searchParams.get("state");
  const error = req.nextUrl.searchParams.get("error");
  if (error) return NextResponse.redirect(new URL(`/login?error=${error}`, req.url));
  if (!code || !state) return NextResponse.redirect(new URL("/login?error=invalid", req.url));

  const store = await cookies();
  const savedState = store.get("oauth_state_naver")?.value;
  const next = store.get("oauth_next")?.value ?? "/";
  if (state !== savedState) {
    return NextResponse.redirect(new URL("/login?error=state_mismatch", req.url));
  }

  // 1) code → access_token
  const tokenRes = await fetch("https://nid.naver.com/oauth2.0/token", {
    method: "POST",
    headers: { "Content-Type": "application/x-www-form-urlencoded" },
    body: new URLSearchParams({
      grant_type: "authorization_code",
      client_id: process.env.NAVER_CLIENT_ID!,
      client_secret: process.env.NAVER_CLIENT_SECRET!,
      code,
      state,
    }),
  });
  const token = await tokenRes.json();
  if (!token.access_token) {
    return NextResponse.redirect(new URL("/login?error=token", req.url));
  }

  // 2) access_token → 사용자 정보
  const profileRes = await fetch("https://openapi.naver.com/v1/nid/me", {
    headers: { Authorization: `Bearer ${token.access_token}` },
  });
  const profile = await profileRes.json();
  // profile.response: { id, email, name, nickname, profile_image, ... }

  // 3) DB upsert (예시)
  // const user = await upsertUser({ provider: "naver", providerId: profile.response.id, ... });

  // 4) 세션 발급
  await createSession({
    userId: profile.response.id,
    provider: "naver",
    email: profile.response.email,
    name: profile.response.name,
  });

  // 5) 임시 쿠키 정리
  store.delete("oauth_state_naver");
  store.delete("oauth_next");

  return NextResponse.redirect(new URL(next, req.url));
}

src/app/api/auth/google/start/route.ts
import { NextRequest, NextResponse } from "next/server";
import { cookies } from "next/headers";
import { randomString, pkceChallenge, TEMP_COOKIE_OPTS } from "@/lib/oauth";

export async function GET(req: NextRequest) {
  const state = randomString();
  const nonce = randomString();
  const verifier = randomString(48);
  const challenge = await pkceChallenge(verifier);
  const next = req.nextUrl.searchParams.get("next") ?? "/";

  const url = new URL("https://accounts.google.com/o/oauth2/v2/auth");
  url.searchParams.set("response_type", "code");
  url.searchParams.set("client_id", process.env.GOOGLE_CLIENT_ID!);
  url.searchParams.set("redirect_uri", process.env.GOOGLE_REDIRECT_URI!);
  url.searchParams.set("scope", "openid email profile");
  url.searchParams.set("state", state);
  url.searchParams.set("nonce", nonce);
  url.searchParams.set("code_challenge", challenge);
  url.searchParams.set("code_challenge_method", "S256");
  url.searchParams.set("access_type", "online"); // refresh_token 필요시 "offline"
  url.searchParams.set("prompt", "select_account");

  const store = await cookies();
  store.set("oauth_state_google", state, TEMP_COOKIE_OPTS);
  store.set("oauth_nonce_google", nonce, TEMP_COOKIE_OPTS);
  store.set("oauth_verifier_google", verifier, TEMP_COOKIE_OPTS);
  store.set("oauth_next", next, TEMP_COOKIE_OPTS);

  return NextResponse.redirect(url);
}

import { NextRequest, NextResponse } from "next/server";
import { cookies } from "next/headers";
import { createSession } from "@/lib/session";

export async function GET(req: NextRequest) {
  const code = req.nextUrl.searchParams.get("code");
  const state = req.nextUrl.searchParams.get("state");
  if (!code || !state) return NextResponse.redirect(new URL("/login?error=invalid", req.url));

  const store = await cookies();
  const savedState = store.get("oauth_state_google")?.value;
  const verifier = store.get("oauth_verifier_google")?.value;
  const next = store.get("oauth_next")?.value ?? "/";
  if (state !== savedState || !verifier) {
    return NextResponse.redirect(new URL("/login?error=state_mismatch", req.url));
  }

  // 1) 토큰 교환
  const tokenRes = await fetch("https://oauth2.googleapis.com/token", {
    method: "POST",
    headers: { "Content-Type": "application/x-www-form-urlencoded" },
    body: new URLSearchParams({
      grant_type: "authorization_code",
      code,
      client_id: process.env.GOOGLE_CLIENT_ID!,
      client_secret: process.env.GOOGLE_CLIENT_SECRET!,
      redirect_uri: process.env.GOOGLE_REDIRECT_URI!,
      code_verifier: verifier,
    }),
  });
  const token = await tokenRes.json();
  if (!token.access_token) {
    return NextResponse.redirect(new URL("/login?error=token", req.url));
  }

  // 2) 사용자 정보 (id_token을 디코드해도 되지만 단순화를 위해 userinfo 사용)
  const userRes = await fetch("https://openidconnect.googleapis.com/v1/userinfo", {
    headers: { Authorization: `Bearer ${token.access_token}` },
  });
  const profile = await userRes.json();
  // profile: { sub, email, email_verified, name, picture, ... }

  await createSession({
    userId: profile.sub,
    provider: "google",
    email: profile.email,
    name: profile.name,
  });

  store.delete("oauth_state_google");
  store.delete("oauth_nonce_google");
  store.delete("oauth_verifier_google");
  store.delete("oauth_next");

  return NextResponse.redirect(new URL(next, req.url));
}

import { NextRequest, NextResponse } from "next/server";
import { cookies } from "next/headers";
import { createSession } from "@/lib/session";

export async function GET(req: NextRequest) {
  const code = req.nextUrl.searchParams.get("code");
  const state = req.nextUrl.searchParams.get("state");
  if (!code || !state) return NextResponse.redirect(new URL("/login?error=invalid", req.url));

  const store = await cookies();

  if (state !== savedState || !verifier) {

  const body = new URLSearchParams({
    grant_type: "authorization_code",

    body,

  const token = await tokenRes.json();
  if (!token.access_token) {
    return NextResponse.redirect(new URL("/login?error=token", req.url));
  }

  // 2) 사용자 정보
  const userRes = await fetch("https://kapi.kakao.com/v2/user/me", {
    headers: { Authorization: `Bearer ${token.access_token}` },
  });
  const profile = await userRes.json();
  // profile: { id, kakao_account: { email, profile: { nickname, profile_image_url } } }

  await createSession({
    userId: String(profile.id),
    provider: "kakao",
    email: profile.kakao_account?.email,
    name: profile.kakao_account?.profile?.nickname,
  });

  store.delete("oauth_state_kakao");
  store.delete("oauth_verifier_kakao");
  store.delete("oauth_next");

  return NextResponse.redirect(new URL(next, req.url));
}

4. 로그인 버튼 컴포넌트(공통)

프론트의 “버튼”은 단순히 /api/auth/{provider}/start로 이동시키는 링크입니다. JS 핸들러나 popup 없이 일반 <a> 태그가 가장 안전합니다(prefetch 방지를 위해 next/link가 아닌 <a> 사용).

5-4. 보호 라우트 — Next.js 16의 proxy.ts

Next.js 16부터 middleware.ts는 deprecated되고 proxy.ts로 이름이 바뀌었습니다. 위치는 src/proxy.ts (또는 프로젝트 루트의 proxy.ts).

src/proxy.ts
import { NextRequest, NextResponse } from "next/server";

const PROTECTED = ["/mypage", "/subscribe"];

export default function proxy(req: NextRequest) {
  const { pathname } = req.nextUrl;
  const isProtected = PROTECTED.some((p) => pathname.startsWith(p));
  if (!isProtected) return NextResponse.next();

  // 쿠키 존재 여부만 체크(낙관적 검사). 실제 검증은 페이지/DAL에서 readSession()으로.
  const hasSession = req.cookies.get("vv_session");
  if (!hasSession) {
    const url = new URL("/login", req.url);
    url.searchParams.set("next", pathname);
    return NextResponse.redirect(url);
  }
  return NextResponse.next();
}

export const config = {
  matcher: ["/((?!api|_next/static|_next/image|favicon.ico|.*\\.png$).*)"],
};

왜 proxy에서 JWT 검증을 하지 않나? proxy.ts는 모든 요청(prefetch 포함)에서 동작하기 때문에 무거운 검증을 하면 성능이 무너집니다. 쿠키 존재 여부만 보는 “낙관적 검사”에 그치고, 실제 신뢰가 필요한 곳(Server Component, Server Action, Route Handler)에서 readSession()으로 다시 검증하세요.

6. Vercel 배포 체크리스트

항목설정 방법
환경 변수 등록 Vercel 프로젝트 → Settings → Environment Variables에 SESSION_SECRET, {NAVER,GOOGLE,KAKAO}_* 모두 등록. Production / Preview / Development 환경 분리 가능.
Redirect URI 일치 각 콘솔(Naver/Google/Kakao)에 등록한 Redirect URI가 Vercel의 실제 도메인과 1자도 다르지 않아야 함. Preview 도메인(*.vercel.app)은 별도 클라이언트를 만들거나, Preview용 서브 콘솔 등록 권장.
NEXT_PUBLIC_APP_URL Production은 https://veggieverse.com, Preview는 비워두거나 VERCEL_URL을 활용. 코드에서 req.url 기반으로 redirect를 만들면 도메인 의존을 줄일 수 있음(이 가이드의 예시들이 그 패턴).
HTTPS 필수 Naver/Google/Kakao 모두 프로덕션 Redirect URI는 https만 허용. Vercel은 자동 HTTPS이므로 별도 설정 불필요.
쿠키 도메인 SameSite=Lax, Secure, HttpOnly를 그대로 둘 것. 서브도메인 간 SSO가 필요하면 domain: ".veggieverse.com" 옵션 추가.
Edge vs Node 런타임 이 가이드의 라우트는 모두 표준 Web API만 쓰므로 둘 다 동작. 단, DB 드라이버에 따라 export const runtime = "nodejs";를 라우트 파일 상단에 추가해야 할 수 있음.

7. 자주 발생하는 오류와 해결

증상원인해결
callback에서 redirect_uri_mismatch 콘솔 등록값과 실제 요청값 불일치 (http vs https, 트레일링 슬래시, 포트 누락) 콘솔의 Redirect URI를 복사·붙여넣기로 .env에 동일하게 입력
state_mismatch 임시 쿠키가 만료(5분 초과)되었거나, 다른 브라우저/시크릿 창에서 콜백을 열었음 처음부터 다시 로그인 시도. 또는 TEMP_COOKIE_OPTS.maxAge를 10분으로 연장
Kakao 토큰 교환 시 KOE320 이미 사용한 code를 재사용 (브라우저가 콜백을 두 번 호출) 콜백 핸들러가 멱등이 되도록 작성, prefetch 유발하는 <Link> 사용 금지
이메일이 undefined로 옴 사용자가 “선택 동의” 항목을 거절 UI에서 “이메일 미제공시 가입 불가” 안내 표시 + 재요청 시 auth_type=reauthorize(Kakao) / prompt=consent(Google) 추가
Next.js 16: cookies() 관련 타입 오류 v15 이전과 달리 cookies()가 Promise 반환 모든 호출부를 const store = await cookies();로 수정
middleware가 동작하지 않음 Next.js 16에서 middleware.ts는 deprecated 파일명을 proxy.ts로 변경, export도 proxy로 변경
Vercel Preview에서 로그인이 깨짐 Preview URL이 매 배포마다 바뀌어 콘솔에 등록 불가 Preview용 별도 OAuth 클라이언트를 만들고 *.vercel.app 또는 고정 Preview 도메인 사용

본 가이드는 Next.js 16.2 기준 공식 문서(node_modules/next/dist/docs/01-app/02-guides/authentication.md)와 각 공급자 OAuth 2.0 사양을 토대로 작성되었습니다. 환경에 맞게 DB upsert·에러 처리·로깅을 보강해 사용하세요.
