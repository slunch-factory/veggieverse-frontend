# Slunch VeggieVerse - 디자인 요청 문서

> 작성일: 2026-04-09
> 목적: 피그마 디자인 작업에 필요한 기능 정의 및 화면 구성 요구사항 정리

---

## 1. 스피릿 (식이유형 설문)

### 개요
- 총 **4단계** 설문 형식
- 타이틀: **"나에게 맞는 음식 유형은?"**
- 서브 카피: "4단계 설문으로 당신의 식이유형, 영양 목표, 알레르기, 매운맛 선호도를 알아보세요."

### 1단계 - 식이유형 선택
- 질문: **"어떤 식이유형을 따르고 계신가요?"**
- 안내 문구: "현재 실천 중이거나 관심 있는 식이유형을 선택해주세요."
- 선택지 (단일 선택):

| 유형 | 설명 |
|------|------|
| 비건 | 동물성 식품을 일체 섭취하지 않아요. |
| 페스코 | 육류는 먹지 않지만 생선과 해산물은 먹어요. |
| 폴로 | 붉은 고기는 먹지 않지만 닭고기와 생선은 먹어요. |

### 2단계 - 영양 목표
- 질문: **"영양 목표는 무엇인가요?"**
- 안내 문구: "중요하게 생각하는 영양 목표를 모두 선택해주세요."
- 선택지 (복수 선택):
  - 저탄수화물
  - 저칼로리
  - 고단백
  - 저나트륨

### 3단계 - 알레르기
- 질문: **"알레르기가 있으신가요?"**
- 안내 문구: "해당하는 항목을 모두 선택해주세요. 없으면 건너뛰셔도 됩니다."
- 선택지 (복수 선택, 스킵 가능):
  - 견과류
  - 땅콩
  - 유제품
  - 복숭아
  - 밀가루

### 4단계 - 매운맛 선호도
- 질문: **"매운맛은 어느 정도 드시나요?"**
- 안내 문구: "선호하는 매운맛 강도를 선택해주세요."
- 선택지 (단일 선택):

| 단계 | 설명 |
|------|------|
| 1단계 | 매운 건 잘 못 먹어요. 순한 맛이 좋아요. |
| 2단계 | 적당히 매운 건 괜찮아요. 중간이 딱 좋아요. |
| 3단계 | 매울수록 좋아요. 얼큰한 맛을 즐겨요. |

### 디자인 고려사항
- 진행 상태 표시 (프로그레스 바 또는 스텝 인디케이터)
- 이전/다음 단계 네비게이션
- 3단계(알레르기)에는 "건너뛰기" 버튼 필요
- 각 단계별 선택 완료 시 시각적 피드백

## 2. 구독

### 2-1. 배송 주기
- 배송 방식: **주기별 일괄 배송** (매일 배송 아님)
- 주기 옵션:

| 주기 | 비고 |
|------|------|
| 3일 | 프리미엄 신선 옵션 |
| 5일 | 프리미엄 신선 옵션 |
| 1주일 | 기본 |
| 2주일 | 기본 |
| 1달 | 기본 |

- 각 배송 주기마다 **가격이 다르게** 표시되어야 함
- 짧은 주기(3일/5일)는 신선도 프리미엄 느낌을 줄 수 있는 시각적 차별화 고려

### 2-2. 결제 옵션
- 결제 수단: 추후 확정 (네이버페이, 카페24 등 검토 중)
- **구독 종료 알림**: 종료 1~2일 전 알림 발송
- **정기 자동 결제**: 선택 가능 (토글 또는 체크박스)

### 2-3. 제품 필터링
- **자동 필터링**: 스피릿에서 설정한 알레르기 재료 포함 제품 자동 제외
- **수동 필터링**: 전체 재료(ingredients) 풀에서 제외하고 싶은 재료를 추가 선택 가능
  - 전체 재료 목록을 나열하고 그 안에서 체크 해제하는 방식

### 2-4. 플랜 종류

| 플랜 | 설명 |
|------|------|
| 디폴트 | 스피릿 기반 랜덤 추천 플랜 |
| 낮은 가격 | 가격 우선 정렬 플랜 |
| 저속 노화 | 안티에이징 중심 식단 |
| 낮은 칼로리 | 칼로리 제한 식단 |
| 커스텀 플랜 | 직접 구성 (지난 플랜 불러오기 기능 포함) |

### 디자인 고려사항
- 주기 선택 시 가격 변동이 실시간으로 반영되는 UI
- 플랜 선택 카드형 레이아웃 권장
- 필터링 설정은 구독 흐름 내에서 자연스럽게 접근 가능하도록
- 커스텀 플랜: 이전 플랜 불러오기 진입점 필요

## 3. 회원가입

### 가입 방식
- **소셜 회원가입**: 구글, 카카오
- **자체 회원가입**: 이메일 기반

### 필요한 사용자 정보
- 이메일
- 비밀번호 (자체 가입 시)
- 닉네임
- (추가 정보는 스피릿 설문으로 수집)

### 디자인 고려사항
- 소셜 로그인 버튼을 상단에 배치하여 간편 가입 유도
- 자체 회원가입 폼은 하단 또는 별도 탭으로 분리
- 가입 완료 후 스피릿 설문으로 자연스럽게 연결되는 흐름

---

## 4. 로그인

### 로그인 방식
- **소셜 로그인**: 구글, 카카오
- **자체 로그인**: 이메일 + 비밀번호

### 로그인 폼 동작
- **다른 페이지에서 열리는 모달/오버레이 형태**
  - 페이지 이동 없이 현재 페이지 위에 로그인 폼이 뜨는 방식
  - 배경 딤(dim) 처리
  - 닫기 버튼 또는 배경 클릭으로 닫기

### 디자인 고려사항
- 모달 형태의 로그인 폼 디자인 필요
- 비밀번호 찾기 링크 포함
- 회원가입 페이지로의 연결 링크 포함
- 모바일에서는 바텀 시트 형태도 고려

---

## 5. 마이페이지

### 탭 네비게이션
- 홈 | 주문 내역 | 레시피 북마크 | 관심상품 | 상품 리뷰 | 회원정보

### 5-1. 홈 탭
#### 프로필 영역
- 프로필 이미지 (기본 아바타)
- 닉네임
- 칭호/뱃지 (예: "Bloomist", MBTI 표시)
- 보유 뱃지 수
- **프로필 수정** 링크

#### 활동 통계
- 3칸 카드 레이아웃:
  - 레시피 수
  - 댓글 수
  - 좋아요 수

#### 최근 주문
- 상단에 "더보기" 링크 (주문 내역 탭으로 이동)
- 주문 항목:
  - 상품명
  - 주문일자
  - 배송 상태 뱃지 (배송완료: 회색 / 배송중: 초록색)
  - 가격

#### 바로가기 메뉴
- 관심상품 (하트 아이콘)
- 레시피 북마크 (북마크 아이콘)
- 상품 리뷰 (말풍선 아이콘)
- 회원정보 (사람 아이콘)
- 각 항목 오른쪽에 chevron(>) 표시

### 5-2. 주문 내역 탭
- 전체 주문 목록
- 주문 상세 보기

### 5-3. 레시피 북마크 탭
- 북마크한 레시피 목록

### 5-4. 관심상품 탭
- 찜한 상품 목록

### 5-5. 상품 리뷰 탭
- 작성한 리뷰 목록

### 5-6. 회원정보 탭
- 개인정보 수정
- 스피릿 재설정 진입점
- 구독 관리
- 로그아웃 / 회원탈퇴

### 디자인 고려사항
- 참고 이미지 기반: 연한 올리브/베이지 톤의 배경색
- 카드형 컴포넌트 활용
- 배송 상태 뱃지 색상 구분 명확하게
- 모바일 반응형 고려

---

## 전체 화면 흐름 (User Flow)

```
회원가입 → 스피릿 설문(4단계) → 홈
                                  ↓
                           구독 페이지 → 플랜 선택 → 배송 주기 선택 → 결제
                                  ↓
                           마이페이지 → 주문 관리 / 구독 관리
```

---

## 디자인 시스템 참고

- **컬러 톤**: 올리브/그린 계열 (자연친화적 이미지)
- **폰트**: 추후 협의
- **반응형**: 데스크톱 + 모바일 대응 필요
- **컴포넌트**: 카드, 뱃지, 모달, 프로그레스 바, 탭 네비게이션 등

---

## 디자인 우선순위

| 순위 | 화면 | 비고 |
|------|------|------|
| 1 | 스피릿 (설문 4단계) | 신규 사용자 온보딩 핵심 |
| 2 | 구독 | 비즈니스 핵심 기능 |
| 3 | 회원가입 / 로그인 | 진입점 |
| 4 | 마이페이지 | 기존 참고 이미지 있음 |
